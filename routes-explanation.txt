# Bitsoft App API Documentation

This document provides a comprehensive guide to the Bitsoft App REST API. All protected routes require a `Bearer <token>` in the `Authorization` header.

---

## 🔐 Authentication (Public)
These endpoints are used for user registration and login.

### Register Student
`POST /api/sign-up`
- **Description:** Registers a new student account.
- **Required Body:**
  - `fullName` (String): Full name of the student.
  - `email` (String): Unique email address.
  - `password` (String): Min 6 characters.
  - `username` (String): Unique username.
  - `parentNumber` (String): Contact number for the parent.
- **Optional Body:**
  - `groupId` (String): ID of the group the student belongs to.

### Login Student
`POST /api/login`
- **Description:** Authenticates a student and returns a JWT token.
- **Required Body:**
  - `email` (String)
  - `password` (String)

### Login Admin
`POST /api/login/admin`
- **Description:** Authenticates an admin/manager.
- **Required Body:**
  - `email` (String)
  - `password` (String)

### Register Admin
`POST /api/sign-up/admin`
- **Description:** Registers a new administrative account.
- **Required Body:**
  - `email` (String)
  - `password` (String)

## 📝 Quiz & Daily Tests (Protected)
Managed via `TestController`. Designed for daily performance tracking.

### Create Batch Tests (Admin Only)
`POST /api/tests/batch`
- **Description:** Schedules tests for multiple dates.
- **Required Body:**
  - `tests` (Array of Tests):
    - `title` (String)
    - `scheduledDate` (Date String - e.g., "2024-04-16")
    - `questions` (Array of Questions):
      - `text` (String): The question body.
      - `points` (Number): Points awarded for correct answer.
      - `options` (Array of Strings): Answer choices.
      - `answer` (String): The correct option matching one entry in `options`.

### Get Today's Test
`GET /api/tests/daily`
- **Description:** Fetches the test scheduled for today.
- **Access:** Student / Admin
- **Returns:** Test structure with questions. Includes `submitted: true/false`.

### Submit Test Answers
`POST /api/tests/submit`
- **Description:** Submits student answers and calculates the final score.
- **Required Body:**
  - `testId` (String): ID of the test being submitted.
  - `answers` (Object): Map of Question IDs to selected options.
    - Example: `{ "question_id_1": "Paris", "question_id_2": "10" }`
- **Access:** Student (One submission per test per student).

### Leaderboard / Ratings
`GET /api/tests/leaderboard`
- **Description:** Returns the ranking of best students based on total quiz points earned.
- **Access:** All Authenticated Users

## 📰 News Management (Admin)
### Create News
`POST /api/news`
- **Required Body:** `title`, `description`
- **Optional Body:** `pictureUrl`, `videoId`, `url`

### Get All News
`GET /api/get-all-news` | `GET /api/get-latest-news`

### Update/Delete News
`PUT /api/news/:newsId` | `DELETE /api/news/:newsId`

---

## 👥 Student & Group Management (Admin)
### Students
- `GET /api/students`: Get all students.
- `GET /api/students/:studentId`: Get detailed profile.
- `PUT /api/students/:studentId`: Update profile.
- `DELETE /api/students/:studentId`: Delete account.

### Groups
- `POST /api/groups`: Create a group. (**Required:** `name`, `courseId`)
- `GET /api/groups`: List all groups.
- `GET /api/groups/:groupId`: Details with linked course and student list.
- `PUT /api/groups/:groupId` | `DELETE /api/groups/:groupId`

---

## 📚 Curriculum & Homework (Admin)
### Courses
- `POST /api/courses`: Create a course. (**Required:** `name`, `description`)
- `GET /api/courses` | `GET /api/courses/:courseId`
- `PUT /api/courses/:courseId` | `DELETE /api/courses/:courseId`

### Homeworks
- `POST /api/homeworks`: Issue homework.
- **Required Body:** `title`, `description`, `deadline`, `studentId`
- **Optional Body:** `checked` (Bool), `point` (Int)
- `GET /api/homeworks`: List all. Use `?studentId=<id>` to filter.
- `PUT /api/homeworks/:homeworkId` | `DELETE /api/homeworks/:homeworkId`

---

## 📈 Student Activities/Ratings (Admin)
### Log Activity
`POST /api/student-activities`
- **Required Body:** `studentId`, `rating`
- **Description:** Logs general behavior or performance rating.

### Manage Activities
- `GET /api/student-activities`: List all or filter by `?studentId=<id>`.
- `GET /api/student-activities/:activityId`: Get single log.
- `PUT /api/student-activities/:activityId` | `DELETE /api/student-activities/:activityId`
